# Инструкция по добавлению новых баз (колод) карточек

В проекте настроена удобная и безопасная (идемпотентная) система добавления новых баз карточек. Вы можете добавлять новые колоды, изменять существующие или обновлять списки карточек без риска сломать текущие данные в базе.

---

## Шаг 1. Создайте PHP-файл со списком карточек

1. Перейдите в директорию `database/seeders/seeds_data/`.
2. Создайте новый PHP-файл (например, `my_new_deck.php`).
3. Файл должен возвращать массив карточек с компактной структурой ключей `q` (вопрос) и `a` (ответ):

```php
<?php

return [
    [
        'q' => 'Слово или вопрос 1',
        'a' => 'Перевод или ответ 1',
    ],
    [
        'q' => 'Слово или вопрос 2',
        'a' => 'Перевод или ответ 2 \n (с транскрипцией на новой строке)',
    ],
];
```

---

## Шаг 2. Размещение и настройка колоды

Регистрация колоды происходит **автоматически** на основе структуры директорий в папке `database/seeders/seeds_data/`.

1. Положите файл колоды в соответствующую папку подкатегории. Например, для английского языка на русском языке интерфейса путь будет:
   `database/seeders/seeds_data/ru/languages/english/my_new_deck.php`
2. Файл должен возвращать массив с метаданными колоды и списком карточек:
   ```php
   <?php

   return [
       'name' => '🚀 Моя Новая Колода', // Название с эмодзи
       'description' => 'Описание этой замечательной колоды', // Краткое описание
       'deck_lang' => 'ru', // Язык интерфейса/подсказок колоды
       'question_lang' => 'ru', // Язык вопросов
       'answer_lang' => 'en', // Язык ответов
       'sort_order' => 15, // [Опционально] Порядковый номер для сортировки. 
                           // Если не указан, колоды внутри подкатегории сортируются по алфавиту имени файла.
       'cards' => [
           ['q' => 'Слово 1', 'a' => 'Перевод 1'],
           ['q' => 'Слово 2', 'a' => 'Перевод 2'],
       ]
   ];
   ```

### 🔀 Управление порядком сортировки (`sort_order`):
* **По умолчанию (без `sort_order`):** Сидер находит все PHP-файлы в подкатегории, сортирует их по алфавиту имени файла и автоматически присваивает им порядковые номера с шагом 10 (`10`, `20`, `30`, `40`...).
* **Кастомный порядок:** Если вам нужно поменять колоды местами или вставить новую колоду между существующими (например, между `10` и `20`), просто добавьте в файл колоды ключ `'sort_order' => 15`. 
* **Безопасность:** Такой способ изменения порядка не требует переименования файлов, поэтому стабильный ID колоды (который генерируется на основе имени файла) и прогресс пользователей в приложении останутся в безопасности.

---

## Шаг 3. Запустите сидер базы данных

Выполните команду сидирования в Docker-контейнере:

```bash
docker compose exec php php artisan db:seed
```

Или для полного сброса базы и заполнения заново:

```bash
docker compose exec php php artisan migrate:fresh --seed
```

### 💡 Полезная информация:
* **Безопасность запуска**: Этот сидер можно запускать сколько угодно раз. Он обновит названия, описания и `sort_order` колод, а также перезапишет шаблонные карточки актуальными данными из файлов в `seeds_data/`, не затрагивая прогресс пользователей.
* **Обновление карт**: Если вы добавили новые карты или исправили опечатки в файле `seeds_data/my_new_deck.php`, просто запустите `db:seed` повторно — изменения сразу применятся в базе шаблонов.
