# Инструкция по добавлению новых баз (колод) карточек

В проекте настроена удобная и безопасная (идемпотентная) система добавления новых баз карточек. Вы можете добавлять новые колоды, изменять существующие или обновлять списки карточек без риска сломать текущие данные в базе.

---

## Шаг 1. Создайте PHP-файл со списком карточек

1. Перейдите в директорию `database/seeders/seeds_data/`.
2. Создайте новый PHP-файл (например, `my_new_deck.php`).
3. Файл должен возвращать массив карточек со структурой `question`, `answer` и `weight` (обычно `0`):

```php
<?php

return [
    [
        'question' => 'Слово или вопрос 1',
        'answer' => 'Перевод или ответ 1',
        'weight' => 0,
    ],
    [
        'question' => 'Слово или вопрос 2',
        'answer' => 'Перевод или ответ 2 \n (с транскрипцией на новой строке)',
        'weight' => 0,
    ],
];
```

---

## Шаг 2. Зарегистрируйте новую колоду в сидере

1. Откройте файл [TemplateDeckSeeder.php](file:///home/artem/mf_new/database/seeders/TemplateDeckSeeder.php).
2. Найдите массив `$decks` в методе `run()`.
3. Добавьте новый элемент в этот массив, указав уникальный `id` (следующий по порядку), метаданные колоды и имя созданного вами файла карточек:

```php
        $decks = [
            // ... существующие колоды ...
            [
                'id' => 8, // Уникальный ID
                'name' => '🚀 Моя Новая Колода', // Название с эмодзи
                'deck_lang' => 'ru', // Язык интерфейса/подсказок колоды
                'question_lang' => 'ru', // Язык вопросов
                'answer_lang' => 'en', // Язык ответов
                'description' => 'Описание этой замечательной колоды', // Краткое описание
                'file' => 'my_new_deck.php', // Имя файла из папки seeds_data
            ],
        ];
```

---

## Шаг 3. Запустите сидер базы данных

Выполните команду сидирования в Docker-контейнере:

```bash
docker compose exec php php artisan db:seed
```

### 💡 Полезная информация:
* **Безопасность запуска**: Этот сидер можно запускать сколько угодно раз. Он обновит названия и описания колод, а также перезапишет шаблонные карточки актуальными данными из файлов в `seeds_data/`, не затрагивая прогресс пользователей.
* **Обновление карт**: Если вы добавили новые карты или исправили опечатки в файле `seeds_data/my_new_deck.php`, просто запустите `db:seed` повторно — изменения сразу применятся в базе шаблонов.
