# Архитектура модуля "Голосовой Урок" (Hands-free Lesson)

Модуль предназначен для изучения карточек без использования рук. Приложение зачитывает вопрос, слушает ответ пользователя через микрофон, оценивает произношение/правильность ответа и зачитывает результат. 

Ниже описана полная архитектура взаимодействия между Flutter-клиентом и Laravel-сервером.

## 1. Фронтенд (Flutter)
Основная логика содержится в `VoiceLessonProvider`, который работает как State Machine.

### Состояния (VoiceLessonState)
* `loading` - загрузка урока и первой карточки
* `playingQuestion` - воспроизведение аудио вопроса (TTS)
* `waitingForSpeech` - запись с микрофона, ожидание ответа пользователя
* `recordingAnswer` - активная запись (когда детектируется голос)
* `uploadingAnswer` - отправка аудиофайла на сервер
* `grading` - ожидание ответа от сервера (проверка ответа)
* `playingFeedback` - озвучивание результата (правильно/неправильно + правильный ответ)
* `finished` - конец урока

### Особенности реализации
* **Запись звука:** Используется пакет `record`.
* **Авто-остановка записи:** Provider постоянно мониторит амплитуду (громкость) звука в децибелах (dB). Если громкость превышает порог (`-65.0 dB`), считается, что пользователь начал говорить. Если после этого наступает тишина на 2.5 секунды, запись автоматически останавливается и отправляется на сервер.
* **Таймаут (Silence Detection):** Если пользователь молчит 15 секунд, срабатывает таймаут, запись прекращается, отправляется пустой запрос, а пользователю зачитывается "Я не расслышал. Правильный ответ: ...".
* **Озвучка (TTS):** Для озвучки используется пакет `just_audio`. Шаблонные фразы ("Отлично", "Неправильно") запрашиваются у бекенда на языке интерфейса пользователя, а сам текст карточки — на изучаемом языке.

## 2. Бекенд (Laravel)

### Контроллеры и Сервисы
* `VoiceLessonController` — принимает запросы `/start` и `/answer`. 
  * Отвечает за маршрутизацию и **очистку текста** (удаление эмодзи, тегов `[v]...[/v]`, скобок `(...)`).
  * Разделяет данные: возвращает оригинальный текст с эмодзи для отображения на экране (`display_question`, `display_answer`) и очищенный текст для проверки и диктора (TTS).
  * Извлекает **только первую строчку** (`getFirstLine`) из ответа для оценки и озвучки, чтобы игнорировать любые дополнительные пояснения.
* `VoiceLessonService` — отвечает за выбор следующей карточки. Использует систему Лейтнера (`progress.next_review_at`).
* `AnswerEvaluationService` — **ядро проверки**.
  * Очищает входящий текст (от Whisper) и правильный ответ от знаков препинания и приводит к нижнему регистру.
  * Сравнивает строки математически (функция `similar_text`).
  * Градация: `>= 90%` = Green (Правильно), `70% - 89%` = Yellow (Почти правильно), `< 70%` = Red (Неправильно).
  * Распознает голосовые команды ("не знаю", "дальше", "пропустить"), чтобы управлять уроком голосом.
* `OpenAiSpeechToTextProvider` — сервис, отправляющий аудио пользователя в **OpenAI Whisper**. Критически важно передавать оригинальное расширение файла (`.m4a` или `.webm`), иначе Whisper выдаст ошибку `Unrecognized file format`.

### Кеширование аудио (AudioService)
Любой текст, который нужно озвучить (вопросы, ответы, шаблонные фразы типа "Отлично, правильно"), проходит через `FlashcardAudioController::directAudio`. 
* Сервер генерирует MD5-хеш от строки `текст + язык`.
* Проверяет таблицу `audio_caches`.
* Если файла нет — делает запрос в Google TTS (или OpenAI TTS), скачивает mp3, кладет на диск и в БД.
* Если файл есть — мгновенно отдает ссылку на закешированный mp3.
* **Итог:** Шаблонные фразы фидбека и ответы кешируются 1 раз и больше не тратят лимиты API.

## 3. Флоу одного цикла (Request/Response Cycle)
1. **Flutter:** Запрашивает аудио вопроса (`/api/flashcards/.../audio/direct?text=Question`).
2. **Flutter:** Включает микрофон. Пользователь говорит ответ.
3. **Flutter:** Отправляет `.webm` (или `.m4a`) файл на `/api/voice-lessons/answer`.
4. **Laravel:** 
   * Передает аудио в OpenAI Whisper.
   * Получает текст.
   * `AnswerEvaluationService` сравнивает текст с ответом карточки (возвращает green/yellow/red).
   * Записывает прогресс (обновляет вес Лейтнера).
   * Достает следующую карточку из БД.
   * Очищает теги и эмодзи.
   * Отвечает JSON-ом с оценкой и данными следующей карточки.
5. **Flutter:** Получает оценку. 
6. **Flutter:** Запрашивает озвучку шаблонной фразы (напр., "Отлично"). Играет её.
7. **Flutter:** Запрашивает озвучку правильного ответа (напр., "Apple"). Играет её.
8. **Flutter:** Переходит к следующей карточке (возврат к шагу 1).

## 4. UI и Управление (Flutter)
* **Настройка чувствительности микрофона:** На стартовом экране реализован ползунок (Slider) в диапазоне от -100 dB (для тишины) до -30 dB (для шумного помещения). Значение динамически обновляет порог срабатывания (`speechAmplitudeThreshold`) и навсегда сохраняется локально на устройстве через `shared_preferences`.
* **Кнопка "Не знаю" (Skip) / Таймаут:** Если пользователь не знает ответа (нажимает кнопку) или молчит 15 секунд, приложение отправляет на сервер POST-запрос с флагом `is_timeout=1`, **не прикрепляя аудиофайл**.
  * Сервер `VoiceLessonController` ожидает параметр `audio` как `nullable`.
  * При отсутствии аудиофайла сервер пропускает запрос к OpenAI Whisper, **экономя AI-токены** и время.
  * Карточка моментально оценивается как Red (Неправильно), пользователю озвучивается правильный вариант, после чего загружается следующая карточка.
* **Распознанный текст (Transcript):** Во время показа результата (фидбека) на экране выводится курсивом текст `You said: '...'`. Это помогает пользователю понять, почему система засчитала ошибку (например, из-за акцента или если микрофон услышал другое слово).

## 5. Затронутые и созданные файлы (History)

В процессе разработки данного модуля были созданы или существенно модифицированы следующие файлы:

### Backend (Laravel)
* `app/Http/Controllers/Api/VoiceLessonController.php` — API-контроллер для старта урока и приема ответов.
* `app/Http/Controllers/Api/FlashcardAudioController.php` — Отдача закэшированного аудио (через Google/OpenAI TTS).
* `app/Services/VoiceLesson/VoiceLessonService.php` — Логика выбора карточек (Leitner).
* `app/Services/VoiceLesson/AnswerEvaluationService.php` — Оценка правильности ответа (математическое сравнение).
* `app/Services/Audio/AudioService.php` — Кеширование аудио.
* `app/Services/Audio/Contracts/SpeechToTextProviderInterface.php` — Интерфейс STT.
* `app/Services/Audio/Providers/OpenAiSpeechToTextProvider.php` — Провайдер OpenAI Whisper.
* `config/audio.php` — Конфиг для аудио сервисов.
* `routes/api.php` — Маршруты `/api/voice-lessons/*`.
* `app/Providers/AppServiceProvider.php` — Регистрация биндингов.

### Frontend (Flutter)
* `mindflasher_4/lib/screens/deck/voice_lesson_screen.dart` — UI голосового урока (кнопки, состояния, анимации).
* `mindflasher_4/lib/providers/voice_lesson_provider.dart` — State Machine логики проведения урока.
* `mindflasher_4/lib/translates/voice_lesson_screen_translate.dart` — Тексты интерфейса.
* `mindflasher_4/lib/screens/deck/deck_card.dart` — Точка входа в урок из списка колод.
* `mindflasher_4/pubspec.yaml` — Подключение зависимостей (`record`, `just_audio` и др.).
