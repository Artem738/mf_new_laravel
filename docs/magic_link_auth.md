# Документация: Вход через мобильный браузер (Magic Link)

Этот модуль разработан для решения проблемы с доступом к микрофону в Telegram Web App (TMA). Из-за жестких ограничений WebView в Telegram для голосовых уроков требовалось постоянное подтверждение пользователя (тап). 
Чтобы решить эту проблему, мы добавили возможность бесшовного перехода из Telegram в полноценный мобильный браузер (Safari/Chrome), где голосовые модули (Web Speech API) работают стабильно и в фоне.

## Как это работает

1. **Генерация ссылки (Backend)**
   В разделе настроек пользователя (во Flutter-приложении внутри Telegram) находится кнопка **"Скопировать ссылку для браузера"**. 
   При нажатии, фронтенд запрашивает бэкенд по защищенному маршруту (`GET /api/user/web-link`). 
   Бэкенд генерирует случайный токен на 64 символа, сохраняет его в базе данных в поле `web_access_key` текущего пользователя и возвращает полную ссылку вида: `https://mf.sitelab-studio.com/a/?key=ОДНОРАЗОВЫЙ_КЛЮЧ`.

2. **Авторизация (Frontend & Backend)**
   Пользователь вставляет эту ссылку в обычный браузер.
   При инициализации (`ProviderUserLogin.initialize()`), Flutter-клиент проверяет параметр `?key=` в URL.
   - Если токен авторизации (Sanctum) **уже сохранен** локально в `FlutterSecureStorage`, приложение **игнорирует ключ** и просто пускает пользователя внутрь. Это решает проблему закладок: сохраненная ссылка со старым ключом будет работать, если сессия еще жива.
   - Если локального токена нет, Flutter делает открытый запрос (`POST /api/web-login`) с передачей `key`. Бэкенд проверяет наличие ключа. Если он валидный, сервер генерирует Sanctum токен, возвращает его клиенту и **мгновенно обнуляет** `web_access_key` в БД. Таким образом, ссылка становится одноразовой и больше не сработает. 
   После получения токена, Flutter сохраняет его локально в безопасное хранилище `DeviceAuthLocalStore`.

## Важная информация
- Базовый URL для генерации ссылки берется из файла `config/app.php` (ключ `web_app_url`). Значение по умолчанию берется из `.env` (ключ `WEB_APP_URL`). Если переменной в `.env` нет, используется дефолтное значение, прописанное в `app.php`.
- Поле `web_access_key` должно быть скрыто (`$hidden`) в модели `User`, чтобы случайно не отдать его в API вместе с профилем.
- Токен `Sanctum` теперь сохраняется локально. Ранее он хранился только в памяти, и авторизация при старте происходила за счет скрытого логина через Telegram. Теперь сессия живет в браузере.

## Затронутые файлы

### Backend (Laravel)
- **`config/app.php`**: Добавлен конфигурационный ключ `web_app_url`.
- **`database/migrations/xxxx_xx_xx_add_web_access_key_to_users_table.php`**: Миграция на добавление колонки `web_access_key`.
- **`app/Models/User.php`**: Поле `web_access_key` добавлено в массивы `$fillable` и `$hidden`.
- **`app/Http/Controllers/UserController.php`**: Метод `generateWebLink` для генерации одноразовой ссылки.
- **`app/Http/Controllers/AuthController.php`**: Метод `webLogin` для обмена ключа на Sanctum-токен.
- **`routes/api.php`**: Добавлены маршруты `/user/web-link` и `/web-login`.

### Frontend (Flutter)
- **`lib/services/auth/auth_api.dart`**: Добавлен метод `loginWithWebKey()`.
- **`lib/providers/provider_user_control.dart`**: Добавлен метод `generateWebLink()` для вызова генерации ссылки.
- **`lib/services/auth/auth_local_store.dart`**: Реализованы методы для сохранения и чтения Sanctum-токена из безопасного хранилища `tokenKey`. 
- **`lib/providers/provider_user_login.dart`**: Добавлена проверка параметра `?key=` при старте приложения, обработка логина и сохранение токена локально (`saveToken`).
- **`lib/screens/user_settings_screen.dart`**: Добавлен UI-элемент (кнопка) для вызова функции генерации и копирования ссылки в буфер обмена.
- **`lib/translates/user_settings_screen_translate.dart`**: Добавлены переводы для кнопки (`copy_web_link`) и уведомления о копировании (`link_copied`).
